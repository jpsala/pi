# Docs

Documentacion viva del workspace `pi`.

## Regla De Lectura Liviana

Para empezar una sesion nueva, leer en este orden:

1. `docs/.generated/context-index.md`.
2. `docs/WORKING_MEMORY.md`.
3. `docs/TOPICS.md`.
4. Topic o track puntual segun el pedido.

No abrir todo `docs/` por defecto.

## Mapa

- `PROJECT.md`: identidad y alcance del proyecto, pendiente de completar.
- `ASSISTANT_RULES.md`: reglas locales para agentes.
- `DEVELOPMENT.md`: stack, comandos y persistencia cuando existan.
- `WORKING_MEMORY.md`: estado vivo corto.
- `TOPICS.md`: router de topics.
- `GLOSSARY.md`: aliases y conceptos.
- `DECISIONS.md`: decisiones durables.
- `OPEN_QUESTIONS.md`: preguntas pendientes.
- `topics/`: runbooks y how-tos recuperables.
- `tracks/`: trabajo vivo retomable.
- `skills/`: skills locales portables.
- `../pi-extensions/README.md`: extensiones/configs Pi propias de JP portables a otras PCs.
- `C:/dev/os/docs/topics/pi-extension-stack.md`: inventario global y flow de herramientas Pi (fuente canonica externa).

## Regla

Guardar valor durable, no transcript. Si el proyecto crece, mover detalle profundo a topics, decisions, tracks o specs.
