# Perfect OS

Auditar y mejorar la calidad agentica de un repo.

Fuente canonica: `docs/topics/os-quality.md`.

## Flujo

1. Abrir la fuente canonica.
2. Revisar por capas: core, docs indexados/livianos, tracks, skills, `.agents`, `.pi`, SpecKit si aplica, scripts, indice y audit.
3. Corregir lo seguro y preguntar por memoria dudosa o cambios de convencion.
4. Regenerar indice y correr audit.
5. Reportar capas OK, cambios, omitidos, pendientes y checks.

## No Hacer

- No tocar producto, datos, deploy ni arquitectura runtime salvo pedido explicito.
- No borrar docs o memoria potencialmente util sin preguntar.
